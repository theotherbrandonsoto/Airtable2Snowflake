Table Relationships:

  • SUBSTANTIATIONS_RAW → SUBSTANTIATIONS_REVISED: One-to-one exact copy relationship
  • SUBSTANTIATIONS_REVISED → SUBSTANTIATIONS_CODED: One-to-many relationship where each code from the JSON array becomes a separate row

Key Points:

  • SUBSTANTIATIONS_RAW is the initial landing table for the data
  • SUBSTANTIATIONS_REVISED is an exact copy of RAW, maintained for data consistency
  • SUBSTANTIATIONS_CODED explodes the Codes JSON array into individual rows with categorization
  • The code_category in SUBSTANTIATIONS_CODED is derived from the code prefix (e.g., "01." → "01. Service and Support Issues")
  • All date fields are stored as VARCHAR in the raw data and should be cast to TIMESTAMP when used
  • Case Number is stored as FLOAT to handle decimal values
